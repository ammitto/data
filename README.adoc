== Purpose

This repository contains the harmonized, deduplicated sanctions dataset that
aggregates data from all Ammitto source repositories into a unified JSON-LD
ontology.

The data is maintained by the Ammitto project and served at www.ammitto.com.

== Data Sources

This harmonized dataset combines data from the following source repositories:

[cols="1,1,3"]
|===
| Source | Code | Description

| EU | `eu` | European Union Consolidated Sanctions List
| UN | `un` | United Nations Security Council Consolidated List
| US | `us` | United States OFAC SDN and Consolidated Lists
| WB | `wb` | World Bank Debarment List
| UK | `uk` | United Kingdom OFSI Consolidated List
| AU | `au` | Australia DFAT Consolidated List
| CA | `ca` | Canada GAC Autonomous Sanctions List
| CH | `ch` | Switzerland SECO Sanctions Database
| CN | `cn` | China MOFCOM/MFA Sanctions Lists
| RU | `ru` | Russia MID/CBR Sanctions Lists
| NZ | `nz` | New Zealand MFAT Russia Sanctions Register
| TR | `tr` | Turkey HMB Asset Freezing Lists
| EU Vessels | `eu_vessels` | EU Designated Vessels (via Denmark DMA)
| UN Vessels | `un_vessels` | UN 1718 Committee Designated Vessels
| JP | `jp` | Japan METI End-User List
|===

== Entity Types

The harmonized ontology supports the following entity types:

PersonEntity:: Individual persons subject to sanctions
OrganizationEntity:: Legal entities, companies, and organizations
VesselEntity:: Ships and maritime vessels
AircraftEntity:: Aircraft subject to sanctions

== Ontology Model

The harmonized data follows an OOP, MECE ontology model implemented in the
ammitto gem. See `lib/ammitto/ontology/` for the Ruby implementation.

=== Class Hierarchy

```
Ontology
├── Types (Enumerations)
│   ├── EntityType (person, organization, vessel, aircraft)
│   ├── IdentificationType (Passport, NationalID, TaxID, etc.)
│   ├── SanctionEffectType (asset_freeze, travel_ban, etc.)
│   └── LegalInstrumentType (regulation, decision, resolution)
│
├── Value Objects (Immutable)
│   ├── NameVariant (full_name, first_name, last_name, script)
│   ├── Address (street, city, region, country, postal_code)
│   ├── Identification (type, number, issuing_country)
│   ├── BirthInfo (date, circa, city, region, country)
│   ├── LegalInstrument (type, identifier, title, url)
│   ├── SanctionEffect (effect_type, scope, description)
│   └── TemporalPeriod (listed_date, effective_date, expiry_date)
│
├── Entities (With Identity)
│   ├── Entity (abstract base)
│   ├── PersonEntity (names, birth_info, gender, nationalities)
│   ├── OrganizationEntity (names, addresses, identifications)
│   ├── VesselEntity (name, imo_number, flag_state, tonnage)
│   └── AircraftEntity (name, serial_number, registration)
│
└── Sanction
    ├── Authority (id, name, country_code, url)
    ├── SanctionRegime (code, name, description)
    ├── SanctionEntry (entity_id, authority, regime, effects, period)
    └── StatusHistory (status, date, reason)
```

== JSON-LD Schema

Each entity follows the Ammitto JSON-LD ontology with the following structure:

[source,json]
----
{
  "@context": "https://www.ammitto.org/ontology/context.jsonld",
  "@id": "https://www.ammitto.org/entity/eu/XX-12345",
  "@type": "PersonEntity",
  "entityType": "person",
  "names": [
    {
      "fullName": "John Doe",
      "isPrimary": true
    }
  ],
  "sourceReferences": [
    {
      "sourceCode": "eu",
      "referenceNumber": "XX-12345"
    }
  ]
}
----

== API Endpoints

The harmonized dataset is available at:

=== Aggregated Files

* **All entities**: `https://www.ammitto.org/api/v1/all.jsonld` (69MB JSON-LD)
* **All entities (Turtle)**: `https://www.ammitto.org/api/v1/all.ttl` (52MB RDF)
* **By source**: `https://www.ammitto.org/api/v1/sources/{source}.jsonld`

=== Knowledge Graph Nodes

* **Entity node**: `https://www.ammitto.org/api/v1/node/entity/{source}/{id}.jsonld`
* **Entry node**: `https://www.ammitto.org/api/v1/node/entry/{source}/{id}.jsonld`
* **Authority node**: `https://www.ammitto.org/api/v1/node/authority/{code}.jsonld`
* **Regime node**: `https://www.ammitto.org/api/v1/node/regime/{code}.jsonld`

=== Search and Facets

* **Search index**: `https://www.ammitto.org/api/v1/search-index.json` (~8MB, 46K entities)
* **Facets**: `https://www.ammitto.org/api/v1/facets/{facet}.json`
  - `authorities.json` - Entity counts by authority
  - `regimes.json` - Entity counts by regime
  - `types.json` - Entity counts by type
  - `countries.json` - Entity counts by country
  - `statuses.json` - Entity counts by status

=== Ontology Browser

* **Classes**: `https://www.ammitto.org/api/v1/ontology/classes.jsonld`
* **Properties**: `https://www.ammitto.org/api/v1/ontology/properties.jsonld`
* **Hierarchy**: `https://www.ammitto.org/api/v1/ontology/hierarchy.json`
* **Examples**: `https://www.ammitto.org/api/v1/ontology/examples/{type}.jsonld`

=== Data Slices

* **By authority**: `https://www.ammitto.org/api/v1/by-authority/{code}.json`
* **By regime**: `https://www.ammitto.org/api/v1/by-regime/{code}.json`
* **By type**: `https://www.ammitto.org/api/v1/by-type/{type}.json`
* **By status**: `https://www.ammitto.org/api/v1/by-status/{status}.json`

== Updating Data

The harmonized dataset is automatically updated via GitHub Actions when
source data repositories are updated.

To manually update:

[source,shell]
----
# From the ammitto gem directory
# Auto-detect and harmonize all data-* repositories
bundle exec ruby exe/ammitto harmonize --scan \
  --sources-dir /path/to/repos \
  --output-dir /path/to/data/api/v1 \
  --verbose

# This generates:
# - Individual node files in node/
# - Search index (search-index.json)
# - Facet counts (facets/*.json)
# - Ontology data (ontology/)
# - Aggregated files (all.jsonld, all.ttl)
----

== Deduplication

The harmonization process includes deduplication logic that:

1. Matches entities across sources using:
   * IMO numbers (for vessels)
   * Name + birth date combinations (for persons)
   * Registration numbers (for organizations)
2. Creates `sameAs` links between matched entities
3. Preserves all source-specific sanction entries
